# 报告生成系统后端详细操作文档

## 一、这个后端是做什么的

本后端用于支撑“报告生成系统”的接口联调，已经按照《前后端接口统一规范文档-二次补充终版》实现了主要接口。

它支持：

- 用户注册、登录、获取当前用户、退出登录
- 普通用户创建报告、生成大纲、保存大纲、流式生成章节内容
- 在线保存章节内容
- 报告记录列表、详情、删除
- DOCX、PDF、Markdown、TXT 多格式导出和下载
- 模板管理、素材管理、模型配置
- 超级管理员用户管理


## 四、启动后的注意事项

后端启动后，PowerShell 窗口要保持打开。

不要关闭这个窗口。

不要按：

```text
Ctrl + C
```

因为按 `Ctrl + C` 会停止后端。

如果你看到下面这种内容：

```text
KeyboardInterrupt
```

说明你刚刚按了 `Ctrl + C`，后端已经被停止，需要重新启动。

## 五、验证后端是否正常

打开浏览器，访问：

```text
http://127.0.0.1:8000/api/health
```

如果看到类似下面的内容，说明后端正常：

```json

  "code": 0,
  "message": "success",
  "data": {
    "status": "ok"
  }
}
```

注意：如果访问下面这个地址：

```text
http://127.0.0.1:8000
```

看到：

```json

  "code": 40400,
  "message": "资源不存在"
}
```

这是正常的。

原因是后端没有做首页，根路径 `/` 不提供页面。真正的接口都在 `/api` 下面。

## 六、前端联调配置

前端的后端基础地址应该填：

```text
http://127.0.0.1:8000
```

不要填：

```text
http://127.0.0.1:8000/api
```

因为前端请求接口时还会继续拼接 `/api/...`。

如果基础地址写成 `http://127.0.0.1:8000/api`，最终可能会变成：

```text
http://127.0.0.1:8000/api/api/auth/login
```

这样接口就会访问失败。

## 八、常用接口地址

### 健康检查

```text
GET /api/health
```

完整地址：

```text
http://127.0.0.1:8000/api/health
```

### 用户登录

```text
POST /api/auth/login
```

请求示例：

```json
{
  "username": "student",
  "password": "123456"
}
```

### 获取当前用户

```text
GET /api/auth/me
```

需要请求头：

```text
Authorization: Bearer <accessToken>
```

### 创建报告

```text
POST /api/reports
```

### 生成大纲

```text
POST /api/reports/{reportId}/outline/generate
```

### 保存大纲

```text
PUT /api/reports/{reportId}/outline
```

### 流式生成报告内容

```text
POST /api/reports/{reportId}/content/generate
```

请求头建议包含：

```text
Accept: text/event-stream
```

### 导出报告

```text
POST /api/reports/{reportId}/exports
```

支持格式：

```text
docx
pdf
md
txt
```

### 下载导出文件

```text
GET /api/reports/{reportId}/exports/{exportId}/download
```

前端下载时要按文件流或 Blob 处理，不要按 JSON 解析。

## 十四、常见问题

### 1. 浏览器访问 `http://127.0.0.1:8000` 显示资源不存在

这是正常的。

后端没有首页，测试后端请访问：

```text
http://127.0.0.1:8000/api/health
```

### 2. PowerShell 出现 `KeyboardInterrupt`

说明你按了：

```text
Ctrl + C
```

后端已经停止，需要重新启动。

### 3. 前端登录失败

检查三件事：

1. 后端 PowerShell 窗口是否还开着。
2. 浏览器访问 `/api/health` 是否成功。
3. 前端后端地址是否是 `http://127.0.0.1:8000`。

### 4. 登录后访问接口提示未登录

检查前端是否带了请求头：

```text
Authorization: Bearer <accessToken>
```

`accessToken` 是登录接口返回的。

### 5. 下载文件失败

下载接口返回的是文件，不是 JSON。

前端需要按 Blob 或文件流处理。

### 6. SSE 流式生成没有显示

这个后端的流式接口是：

```text
POST /api/reports/{reportId}/content/generate
```

原生 `EventSource` 默认只支持 GET，不适合直接请求这个 POST 接口。

前端建议使用：

```text
fetch 流式读取
```

或者：

```text
fetch-event-source
```

## 十五、推荐联调顺序

建议前端按这个顺序联调：

```text
1. 健康检查 /api/health
2. 登录 /api/auth/login
3. 获取当前用户 /api/auth/me
4. 创建报告 /api/reports
5. 生成大纲 /api/reports/{reportId}/outline/generate
6. 保存大纲 /api/reports/{reportId}/outline
7. 流式生成内容 /api/reports/{reportId}/content/generate
8. 查询报告详情 /api/reports/{reportId}
9. 保存章节内容 /api/reports/{reportId}/chapters/{chapterId}/content
10. 创建导出 /api/reports/{reportId}/exports
11. 下载导出文件 /api/reports/{reportId}/exports/{exportId}/download
12. 报告记录列表 /api/reports
13. 管理端模板、素材、模型配置接口
14. 超级管理员用户管理接口
```
